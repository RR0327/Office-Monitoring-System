# Hardware Subsystem Architecture & Simulation Blueprint

This directory contains the engineering specifications, wiring logic, and firmware implementations for the hardware abstraction layer of the **Smart Office Monitoring System**.

## 1. Design Philosophy & Safety Isolation

Microcontrollers operating on low-power DC rails ($3.3\text{V}$ / $5\text{V}$) cannot directly toggle mains-level electrical components without systemic failure. This configuration establishes a realistic physical topology using a two-tier architectural approach:

- **Isolation Layer:** Optocoupled relays ensure complete galvanic isolation, separating the logic processing unit from the high-voltage $220\text{V}$ AC distribution lines.
- **Active Monitoring (Sensing):** Rather than blindly tracking arbitrary software state variables, an **ACS712 Hall-Effect current sensor IC** is placed inline with the active branch of the AC grid, feeding physical feedback to the control loop.

## 2. Core Subsystem Workflow

1. **Downstream Commands (Control):** The central Django event-driven scheduling pipeline sends state configurations down the UART serial loop utilizing the format `DEVICE_ID:STATE` (e.g., `DR_FAN1:1`).
2. **Upstream Telemetry (Sensing):** The firmware systematically monitors the ADC voltage changes generated by the simulated current sensor, processes active power metrics, and continuously fires standard JSON-parsable payload objects upstream across the serial interface.
