# Hardware Overview

## Project
**Office Monitoring System**

## Target Hardware
ESP32-WROOM-32E Development Board

## Scope

This hardware schematic represents a **single office room (Drawing Room)** as a conceptual prototype for the Office Monitoring System.

The complete office contains three identical rooms, each consisting of:

- 2 Fans
- 3 Lights

To avoid unnecessary duplication, only one representative room is implemented in Wokwi, as permitted by the hackathon requirements. The same hardware design can be replicated for the remaining rooms.

---

# Hardware Design

The ESP32 controls five simulated electrical devices:

- Fan 1
- Fan 2
- Light 1
- Light 2
- Light 3

For simulation purposes, **LEDs are used to represent both fans and lights**.

- **Yellow LEDs** represent Fans.
- **Blue LEDs** represent Lights.

Each LED is connected to an ESP32 GPIO pin through a 220Ω current-limiting resistor.

Although the simulation directly drives LEDs, in a real-world deployment each GPIO would instead control a relay module responsible for switching the corresponding 220V AC electrical load.

---

# Pin Mapping

| ESP32 Pin | Connected Component | Represents | Real-World Function |
|------------|---------------------|------------|---------------------|
| GPIO13 | Yellow LED | Drawing Room – Fan 1 | Represents Fan 1 control (Relay in real deployment) |
| GPIO12 | Yellow LED | Drawing Room – Fan 2 | Represents Fan 2 control (Relay in real deployment) |
| GPIO14 | Blue LED | Drawing Room – Light 1 | Represents Light 1 control (Relay in real deployment) |
| GPIO27 | Blue LED | Drawing Room – Light 2 | Represents Light 2 control (Relay in real deployment) |
| GPIO26 | Blue LED | Drawing Room – Light 3 | Represents Light 3 control (Relay in real deployment) |
| GND | Common Ground | All LEDs | Shared ground reference |

---

# Simulation Logic

The ESP32 controls each GPIO independently.

When a GPIO outputs a HIGH signal:

- The corresponding LED turns ON.
- This represents the associated fan or light being switched ON.

When a GPIO outputs a LOW signal:

- The LED turns OFF.
- This represents the device being switched OFF.

The hardware schematic is intended to demonstrate the physical control concept only. The actual device states displayed on the web dashboard and Discord bot are generated by the Django-based simulator and synchronized through the shared backend.

---

# Real-World Deployment

In an actual installation, the LEDs would be replaced with relay modules connected to the office's electrical wiring.

The intended control path would be:

ESP32 GPIO
→ Relay Module
→ AC Fan / Light

An optional ACS712 current sensor could also be added to monitor real-time electrical current for accurate power consumption measurement.
